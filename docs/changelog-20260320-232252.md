# 变更说明：动态 AI 设置 & DeepSeek 集成

**日期**：2026-03-20 23:22  
**涉及功能**：AI 设置管理、DeepSeek API 接入、UI 集成  

---

## 一、变更概述

本次变更将原本硬编码的 AI 桩函数（stub）替换为真实的 DeepSeek / OpenAI 兼容 API 调用，并新增了一套完整的用户可配置设置系统。应用开箱即用——通过环境变量提供默认 API Key，用户也可在界面中自行修改。

---

## 二、新增文件

### 1. `src/hooks/use-settings.ts` — AI 设置状态管理 Hook

- 定义 `AISettings` 接口，包含三个字段：
  - `apiKey`：默认取自 `process.env.NEXT_PUBLIC_DEFAULT_API_KEY`
  - `baseUrl`：默认 `"https://api.deepseek.com"`
  - `modelName`：默认 `"deepseek-chat"`
- 使用 LocalStorage key `kb-ai-settings` 持久化
- 暴露三个方法：
  - `settings`：当前设置对象
  - `updateSettings(partial)`：部分更新并持久化
  - `resetSettings()`：清除 LocalStorage，恢复环境变量默认值

### 2. `src/components/settings-dialog.tsx` — 设置弹窗组件

- 基于 shadcn/ui `Dialog` 组件构建
- 包含三个输入框：
  - API Key（`type="password"`，密码遮罩显示）
  - Base URL（文本输入）
  - Model Name（文本输入）
- 底部按钮：
  - 「恢复默认」：调用 `resetSettings()` 清除自定义配置
  - 「保存」：调用 `updateSettings()` 写入 LocalStorage
- 每次打开时同步最新 settings 到本地 state

### 3. `.env.local` — 环境变量模板

- 包含 `NEXT_PUBLIC_DEFAULT_API_KEY` 占位符
- 已被 `.gitignore` 中的 `.env*` 规则排除，不会被提交到仓库

## 三、修改文件

### 4. `src/lib/ai-service.ts` — AI 服务层重写

**改动前**：`callAI` 是一个 stub（模拟延迟后返回空字符串），`generateTags` 和 `generateSummary` 使用纯本地关键词提取 / 首句截取。

**改动后**：

| 函数 | 说明 |
|------|------|
| `callAI(prompt, settings)` | 真实的 `fetch` 调用 `{baseUrl}/v1/chat/completions`，发送 OpenAI 兼容格式请求 |
| `generateTags(content, settings)` | Prompt AI 返回 3-5 个关键词的 JSON 数组；若无 API Key 或调用失败，自动降级为本地 `extractKeywords` |
| `summarizeNote(content, settings)` | Prompt AI 返回 1 句话摘要；降级逻辑同上 |
| `localGenerateTags(content)` | 导出的本地标签生成函数，供防抖自动标签使用 |
| `findRelatedNotes(noteId, allNotes)` | 未改动，仍使用本地关键词重叠算法 |

**降级策略**：
- `apiKey` 为空 → 直接走本地逻辑，不发请求
- API 请求失败（网络错误 / 非 200 状态码）→ `catch` 后静默降级为本地逻辑
- AI 返回格式异常 → 解析失败后降级

### 5. `src/components/sidebar.tsx` — 侧边栏

**新增内容**：
- 底部工具栏新增齿轮图标按钮（`lucide-react` 的 `Settings` 图标）

**位置**：齿轮按钮位于笔记数量右侧、语言切换和主题切换按钮左侧。

### 6. `src/components/note-editor.tsx` — 笔记编辑器

**新增内容**：
- 新增 `settings: AISettings` prop
- 新增 `isTagging` 状态（控制 AI Tag 按钮的 loading）
- 工具栏右侧新增「AI Tag」按钮：
  - 点击调用 `generateTags(content, settings)` 进行真实 AI 标签生成
  - 加载中显示 `Loader2` 旋转动画
- 原「Summarize」按钮改为调用 `summarizeNote(content, settings)`

**行为变化**：
- **防抖自动标签**（打字时自动触发）：改为使用 `localGenerateTags()`，不调用 AI API，避免浪费额度
- **Cmd/Ctrl+S 保存**：同上，使用本地标签
- **「AI Tag」按钮**：用户主动点击时才调用真实 AI API
- **「Summarize」按钮**：用户主动点击时调用真实 AI API

### 7. `src/app/page.tsx` — 主页面

## 四、架构数据流

```
环境变量 NEXT_PUBLIC_DEFAULT_API_KEY
        │
        ▼
  useSettings Hook ◄──► LocalStorage (kb-ai-settings)
        │
        ├──► Sidebar → SettingsDialog (用户配置 UI)
        │
        └──► NoteEditor
              ├── "AI Tag" 按钮 → generateTags(content, settings) → callAI → DeepSeek API
              └── "Summarize" 按钮 → summarizeNote(content, settings) → callAI → DeepSeek API
              │
              └── 防抖自动标签 → localGenerateTags(content)  [纯本地，不调 API]
```

## 五、API Key 配置方式

### 方式 A：环境变量（部署/默认值）

在项目根目录 `.env.local` 中设置：

```
NEXT_PUBLIC_DEFAULT_API_KEY=sk-你的密钥
```

- 本地开发：修改后需重启 `next dev`
- Vercel 部署：在 Dashboard → Settings → Environment Variables 中添加

### 方式 B：界面配置（用户自定义）

1. 点击侧边栏底部的齿轮图标
2. 在弹窗中填写 API Key、Base URL、Model Name
3. 点击「保存」

界面配置保存在浏览器 LocalStorage 中，**优先级高于环境变量默认值**。点击「恢复默认」可清除自定义配置，回到环境变量值。
