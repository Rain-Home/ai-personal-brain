## 一、背景与目标

本次工作包含两个紧密关联的需求：

### 需求 A：冷启动种子数据

新用户（或评审）首次打开应用时，`localStorage` 为空，看到的是空白页面。无法体验到「结构化 AI 标签」和「语义关联推荐」两大核心能力。

**目标：** 检测到空数据状态时，自动注入两篇高质量示例笔记，使两大引擎立即可演示。

### 需求 B：移除自动标签，改为手动标签

原有的 `localGenerateTags` 基于词频提取关键词并自动覆盖 `tags` 字段，产出的标签噪声大、质量低。

**目标：** 移除自动标签行为，改由用户手动添加/删除标签。同时提升手动标签在推荐引擎中的权重，使其成为高意图信号。

## 二、修改清单

### 1. 新增文件 — `src/constants/seed-data.ts`

定义并导出 `INITIAL_NOTES` 数组（类型为 `Omit<Note, "id" | "createdAt" | "updatedAt">[]`），包含两篇种子笔记：

| 笔记 | 标题 | 内容来源 | 用户标签 | AI 标签 |
|------|------|----------|----------|---------|
| Note 1 | 🚀 Core Feature: Structured AI Tags & Semantic Engine | `changelog-20260321-202446.md` 全文 | Refactor, AI-PM, Architecture | Knowledge Graph (domain/1.0), Heuristic Scoring (tech/0.9), System Design (action/0.8) |
| Note 2 | 📅 Development Milestone: From MVP to Intelligent Brain | 手写项目三阶段演进总结 | Roadmap, AI-PM, Architecture | Knowledge Graph (domain/0.8), Product Roadmap (action/0.7) |

**设计要点：**

- `id`、`createdAt`、`updatedAt` 不硬编码——在注入时由 `storage.ts` 动态生成
- `summary` 和 `conceptSummary` 均已预填充，展现完整的 "AI 增强" 外观
- 两篇笔记共享 AI 标签 `Knowledge Graph` 和用户标签 `AI-PM`、`Architecture`，确保关联推荐引擎能立即触发

**预期关联分数：**

```
AI 标签 "Knowledge Graph" 匹配：+10
分类共现（domain + action）：  +10
用户标签 "AI-PM" 匹配：       +8
用户标签 "Architecture" 匹配：  +8
────────────────────────────────
总分：36
推荐理由："Both notes focus on #Knowledge Graph and #AI-PM and #Architecture"
```

### 2. 存储层扩展 — `src/lib/storage.ts`

**新增常量：**

| 常量 | 值 | 用途 |
|------|----|------|
| `ONBOARDED_KEY` | `"kb-onboarded"` | localStorage 标记位，防止用户清空笔记后重复注入 |

**新增函数 `seedInitialNotes()`：**

- 检查 `kb-onboarded` 标记位——若已存在则返回空数组
- 为每条种子笔记生成 `crypto.randomUUID()` 作为 `id`，`new Date().toISOString()` 作为时间戳
- 写入 `localStorage` 并设置标记位

**`getNotes()` 流程变更：**

```
getNotes()
  │
  ├─ typeof window === "undefined" → return []
  │
  ├─ migrateNotes()
  │
  ├─ localStorage 无 kb-notes 键 → seedInitialNotes()
  │
  ├─ JSON.parse 成功
  │    ├─ 数组为空 → seedInitialNotes()
  │    └─ 数组非空 → 返回
  │
  └─ JSON.parse 失败 → return []
```

### 3. AI 服务层清理 — `src/lib/ai-service.ts`

#### 3a. 移除 `localGenerateTags` 导出

| 维度 | 改造前 | 改造后 |
|------|--------|--------|
| `localGenerateTags` | 导出函数，被 `note-editor.tsx` 调用 | **删除** |
| `extractKeywords` | 私有函数 | 保持不变（`findRelatedNotes` 关键词降级仍需使用） |
| `localFallbackTags` | 内部调用 `localGenerateTags` | 直接调用 `extractKeywords`，不再依赖已删除的函数 |

#### 3b. `localFallbackTags` 重写

```typescript
// 改造前
function localFallbackTags(content: string): GenerateTagsResult {
  const keywords = localGenerateTags(content);  // ← 依赖已删除的函数
  ...
}

// 改造后
function localFallbackTags(content: string): GenerateTagsResult {
  const keywords = extractKeywords(content.trim()).slice(0, 5);  // ← 直接调用
  ...
}
```

#### 3c. `findRelatedNotes` 权重调整

| 评分维度 | 改造前 | 改造后 | 理由 |
|----------|--------|--------|------|
| 用户标签匹配 | +2 分/个 | **+8 分/个** | 标签现在是用户手动添加的高意图信号 |
| 用户标签纳入推荐理由 | 不纳入 | **纳入 `matchedLabels`** | 在推荐理由字符串中展示匹配的用户标签 |

AI 标签匹配（+10 分/个）和分类共现（+5 分/类）保持不变。

### 4. 编辑器组件重构 — `src/components/note-editor.tsx`

#### 4a. 移除自动标签机制

| 移除项 | 说明 |
|--------|------|
| `import { localGenerateTags }` | 不再使用 |
| `import { useDebounce }` | 不再使用（该 Hook 仅此组件使用） |
| `debouncedTitle` / `debouncedContent` | 替换为 `useRef` + `setTimeout` 的手动防抖 |
| 防抖 `useEffect` 中的 `localGenerateTags` 调用 | 防抖现在仅保存 `{ title, content }`，不触碰 `tags` |
| Cmd/Ctrl+S 中的 `localGenerateTags` 调用 | 同上，仅保存 `{ title, content }` |

#### 4b. 新增手动标签输入 UI

在右侧面板「Tags」区域，替换原来的只读 Badge 列表为交互式组件：

**新增状态：** `tagInput: string`

**新增导入：** `Input`（shadcn/ui）、`X`（lucide-react）、`useRef`

**交互逻辑：**

| 操作 | 行为 |
|------|------|
| 输入框按 Enter 或逗号 | 去除首尾空格和逗号 → 大小写不敏感去重 → 追加到 `note.tags` → 调用 `onUpdate` 持久化 → 清空输入框 |
| 点击标签上的 ✕ 按钮 | 从 `note.tags` 中过滤掉该标签 → 调用 `onUpdate` 持久化 |

**UI 结构：**

```
┌─ Tags 区域 ──────────────────┐
│  [Refactor ✕] [AI-PM ✕]     │
│  [Architecture ✕]            │
│  ┌──────────────────────┐    │
│  │ Add tag...           │    │
│  └──────────────────────┘    │
└──────────────────────────────┘
```

每个标签使用 `Badge variant="secondary"` 渲染，右侧内嵌一个圆形按钮包含 `X` 图标。输入框使用 `Input` 组件，高度 `h-7`，字号 `text-xs`。

## 三、数据流总览

```
用户编辑笔记
  │
  ├─ 防抖 500ms ──→ onUpdate({ title, content })
  │                        ↓
  │                   仅更新标题和内容，不触碰 tags
  │
  ├─ Cmd/Ctrl+S ──→ onUpdate({ title, content })
  │                        ↓
  │                   同上
  │
  ├─ 输入标签 + Enter/逗号
  │       ↓
  │     onUpdate({ tags: [...existing, newTag] })
  │
  ├─ 点击标签 ✕
  │       ↓
  │     onUpdate({ tags: filtered })
  │
  └─ 点击 "AI Tag" 按钮
       │
       ↓
     generateTags(content, settings)
       │
       ├─ 有 API Key ──→ 调用 LLM ──→ 解析 JSON ──→ { aiTags, conceptSummary }
       │                                   │
       │                              解析失败 → localFallbackTags()
       │
       └─ 无 API Key ──→ localFallbackTags()
       │
       ↓
     onUpdate({ aiTags, conceptSummary })
                    ↓
              仅更新 aiTags + conceptSummary，不触碰 tags


首次启动（冷启动）：
  getNotes()
       │
       ├─ localStorage 为空 或 notes.length === 0
       │    ├─ kb-onboarded 存在 → return []
       │    └─ kb-onboarded 不存在 → seedInitialNotes() → 注入 2 篇种子笔记
       │
       └─ localStorage 有数据 → 正常返回


关联推荐计算：
  findRelatedNotes(noteId, allNotes)
       │
       ↓
     对每条其他笔记计算加权得分：
       AI 标签名匹配  → +10 分/个（含推荐理由）
       分类共现       → +5 分/类
       用户标签匹配   → +8 分/个（含推荐理由）★ 权重提升
       (降级) 关键词重叠 → +1 分/词
       │
       ↓
     排序取 Top 5 → 附带 reason 说明 → 渲染到 UI
```
